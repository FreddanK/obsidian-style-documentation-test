Obsidian is a local-first Markdown note-taking application built around a personal knowledge graph. Notes are stored as plain `.md` files on disk, which makes them portable, version-controllable, and editor-agnostic.

## Why Obsidian for team documentation

- Notes are plain Markdown files — no proprietary format or vendor lock-in.
- [[Wikilinks]] create a navigable knowledge graph out of flat files.
- The graph view lets contributors discover unexpected connections between topics.
- Compatible with [[Git]] for version control and collaborative review via pull requests.
- Can be published as a static website with [[Quartz]] or [[Docusaurus]].

## Key concepts

- [[Wikilinks]] — the primary navigation mechanism.
- [[Maps Of Content]] — curated index pages for discoverability.
- [[Obsidian best practices]] — conventions for this vault.
- [[Obsidian shared settings]] — how shared configuration is managed across contributors.

## Plugins used

- **Git** (`obsidian-git`) — version control integration inside Obsidian.
- **Excalidraw** (`obsidian-excalidraw-plugin`) — embed diagrams directly in notes. See [[Excalidraw]].

## Contributor paths

There are three levels of engagement with this documentation:

| Role | Tool | Notes |
|------|------|-------|
| Reader | Browser (published site) | No setup needed |
| Occasional contributor | GitHub web editor | Edit `.md` files directly on GitHub |
| Regular contributor | Obsidian + Git | Full experience; follow [[Obsidian shared settings]] setup |

See [[CONTRIBUTING]] for step-by-step instructions for each path.
